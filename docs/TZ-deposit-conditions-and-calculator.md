# ТЗ: Гибкая модель условий вклада (мультивалютность, срок/сумма → ставка)

## 1. Цель

В отличие от кредита с фиксированными условиями (одна ставка, одна макс. сумма, один срок), вклад имеет:

- **Несколько валют** (рубли, доллары, евро и т.д.) — у каждой свои ставки и диапазоны сумм.
- **Зависимость ставки от срока и суммы**: при одном сроке — одна сетка (сумма от–до → ставка), при другом сроке — другая (пример: 31 день 15 000–100 000 $ → 0.05%, 31 день от 100 000 $ → 0.1%; 1095 дней те же суммы → 1% и 3.1%).

Все числовые условия должны храниться в **отдельных полях/таблицах**, а не в тексте описания, чтобы:

- выводить их на странице вклада в виде таблиц;
- использовать в **калькуляторе** (подстановка суммы/срока/валюты → получение ставки и расчёт дохода).

---

## 2. Текущее состояние проекта

- **Модель:** `App\Models\Deposit` — плоские поля: `rate`, `min_rate`, `max_rate`, `term_months`, `min_term_days`, `max_term_days`, `min_amount`, `max_amount`, булевы (капитализация, пополнение и т.д.), `description`.
- **Страница вклада:** `resources/views/deposits/show.blade.php` → компонент `deposit-offer-card.blade.php` — выводит одну ставку, один срок, одну пару мин/макс сумма (фактически в рублях).
- **Админка:** `App\Filament\Resources\Deposits\Schemas\DepositForm` — редактирование плоских полей.

Требуется расширить модель и интерфейсы без полного отказа от текущих полей (обратная совместимость для списков и фильтров).

---

## 3. Предлагаемая модель данных

### 3.1. Варианты хранения

| Вариант | Плюсы | Минусы |
|--------|--------|--------|
| **A. Отдельные таблицы** (валюта + условия) | Удобные запросы, нормализованные данные, простой расчёт в калькуляторе | Больше миграций и моделей |
| **B. JSON в полях deposit** | Быстро внедрить, гибкая схема | Сложнее выборки и индексы, калькулятор парсит JSON |

**Рекомендация:** вариант A — отдельные таблицы, чтобы калькулятор и фильтры могли опираться на нормализованные поля.

### 3.2. Структура (вариант A)

**Таблица `deposits`** — без удаления существующих полей. Оставить: `rate`, `min_rate`, `max_rate`, `term_months`, `min_term_days`, `max_term_days`, `min_amount`, `max_amount` и все булевы. Их можно заполнять при сохранении из новой структуры (агрегаты по всем валютам) для списков и фильтров.

**Таблица `deposit_currencies`**

- `id`
- `deposit_id` (FK → deposits)
- `currency` — код валюты: `RUB`, `USD`, `EUR` (строка, 3–5 символов)
- `min_amount` — мин. сумма по этой валюте (nullable, decimal)
- `max_amount` — макс. сумма по этой валюте (nullable, decimal)
- `sort_order` — порядок вывода (int, default 0)
- `timestamps`

Один вклад может иметь несколько записей в `deposit_currencies` (рубли, доллары, евро).

**Таблица `deposit_conditions`** (или `deposit_rates`)

Одна запись = одна «ячейка»: при таком сроке и таком диапазоне суммы — такая ставка.

- `id`
- `deposit_currency_id` (FK → deposit_currencies)
- `term_days_min` — срок от (дней), nullable
- `term_days_max` — срок до (дней), nullable. Если оба заданы — диапазон; если только один — фиксированный срок
- `amount_min` — сумма от (decimal), nullable
- `amount_max` — сумма до (decimal), nullable
- `rate` — годовая ставка в процентах (decimal)
- `sort_order` — порядок вывода (int, default 0)
- `timestamps`

Пример для «Максимальный в долларах»:

- Валюта: USD, min_amount 15 000, max_amount без ограничений (или null).
- Условия:
  - term 31, amount 15 000–100 000 → rate 0.05
  - term 31, amount от 100 000 → rate 0.1
  - term 1095, amount 15 000–100 000 → rate 1
  - term 1095, amount от 100 000 → rate 3.1

Калькулятор: по выбранной валюте, сроку и сумме находит подходящую запись (срок в диапазоне, сумма в диапазоне) и берёт `rate`.

### 3.3. Модели Eloquent

- **DepositCurrency** — `belongsTo(Deposit::class)`, `hasMany(DepositCondition::class)`.
- **DepositCondition** — `belongsTo(DepositCurrency::class)`.
- **Deposit** — `hasMany(DepositCurrency::class)` с `orderBy('sort_order')`.

Типы полей и `fillable`/`casts` задать в соответствии с миграциями (decimal, integer, string).

## 4. Админка (Filament)

### 4.1. Редактирование вклада

- Сохранить текущие поля формы (название, банк, тип вклада, булевы, описание и т.д.).
- Добавить блок **«Условия по валютам»** (Repeater или отдельная связь):

  1. **Повторяемая группа «Валюта»:**
     - Выбор валюты (Select: RUB, USD, EUR или справочник).
     - Мин. сумма по валюте (опционально).
     - Макс. сумма по валюте (опционально).
     - Сортировка (sort_order).

  2. **Внутри каждой валюты — повторяемая группа «Условие» (ставка по сроку/сумме):**
     - Срок от (дней), срок до (дней) — оба опционально; если заполнен только один, считать фиксированным сроком.
     - Сумма от, сумма до (в выбранной валюте).
     - Ставка (% годовых).
     - Сортировка.

- Валидация: в рамках одной валюты не допускать пересекающихся диапазонов сроков/сумм с разными ставками, если это противоречит бизнес-логике (по желанию заказчика можно сначала без жёсткой проверки пересечений).
- При сохранении: синхронизировать `deposit_currencies` и `deposit_conditions` (удалить лишние, обновить/создать по данным формы). При отсутствии валют оставить текущее поведение (плоские поля).

### 4.2. Обратная совместимость в админке

- Поля «Мин. сумма», «Макс. сумма», «Ставка», «Срок» и т.д. на уровне вклада оставить. Их можно:
  - заполнять автоматически из новой структуры при сохранении (например, макс. ставка по всем условиям, макс. срок, глобальные мин/макс суммы по RUB), **или**
  - оставить ручной ввод только для списков/фильтров, если «условия по валютам» не заданы.

## 5. Публичная страница вклада

### 5.1. Контроллер

- В `DepositController::show` подгружать связь: `Deposit::with(['bank', 'currencies.conditions'])` (или `deposit_currencies`/`deposit_conditions` в зависимости от имён связей).
- Передавать в представление один объект `$deposit` с подгруженными валютами и условиями.

### 5.2. Шаблон и компонент

- **Общие блоки** (название, банк, тип вклада, булевы, описание) — оставить как сейчас.
- **Блок «Условия по валютам»** (новый):
  - Цикл по `$deposit->currencies` (или `depositCurrencies`).
  - Для каждой валюты:
    - Заголовок: валюта (RUB / USD / EUR и подпись «руб.», «долл.» и т.д.).
    - При необходимости — мин/макс сумма по валюте.
    - **Таблица условий**: столбцы, например, «Срок (дней)», «Сумма от», «Сумма до», «Ставка (% годовых)». Строки — из `currency->conditions`, отсортированные по `sort_order` (и при желании по сроку/сумме).
  - Суммы и ставки выводить в читаемом формате (пробелы как разделители тысяч, единый формат процентов).
- Если у вклада нет записей в `deposit_currencies`, показывать текущий блок с одной ставкой/сроком/суммой из плоских полей (как сейчас в `deposit-offer-card`), чтобы старые вклады продолжали отображаться.

### 5.3. Данные для калькулятора

- Либо вывести в разметку JSON с массивом условий (по валютам) в `<script type="application/json" id="deposit-conditions">` и использовать его в JS-калькуляторе.
- Либо сделать отдельный API-эндпоинт, например `GET /api/deposits/{id}/conditions?currency=RUB`, возвращающий структурированный список условий (сроки, диапазоны сумм, ставки) для выбранной валюты.

Оба подхода позволяют не дублировать логику: данные всегда берутся из БД (поля/таблицы), а не из описания.

## 6. Калькулятор (последующая реализация)

- Входные данные: вклад (id), валюта, срок (дней или даты начала/окончания), сумма.
- Алгоритм: по валюте выбрать `deposit_currency`, по сроку и сумме найти подходящую запись в `deposit_conditions` (срок в [term_days_min, term_days_max] или равен одному из них; сумма в [amount_min, amount_max]).
- По найденной `rate` посчитать доход (по формуле начисления процентов: простая/с капитализацией — в зависимости от флага вклада).
- Реализация: либо только фронт (данные из JSON на странице), либо бэкенд-метод/API, возвращающий ставку и при необходимости рассчитанный доход.

## 7. Этапы внедрения (рекомендуемый порядок)

1. **Миграции:** создать таблицы `deposit_currencies` и `deposit_conditions`, не трогая существующие поля в `deposits`.
2. **Модели:** `DepositCurrency`, `DepositCondition`, связи в `Deposit`.
3. **Админка:** форма вклада — добавить Repeater (или вложенный Repeater) для валют и условий; сохранение/обновление/удаление связанных записей.
4. **Синхронизация плоских полей (опционально):** при сохранении вклада заполнять `rate`, `min_rate`, `max_rate`, `term_months`, `min_amount`, `max_amount` из условий (например, по RUB или максимумы по всем валютам) для обратной совместимости списков и фильтров.
5. **Публичная страница:** обновить `deposit-offer-card` (или вынести блок в отдельный компонент): при наличии валют выводить блок «Условия по валютам» с таблицами; иначе — текущий вид по плоским полям.
6. **Данные для калькулятора:** добавить вывод JSON на страницу и/или API для условий.
7. **Калькулятор:** отдельная задача — UI и логика расчёта по выбранной валюте/сроку/сумме.

---

## 8. Важные замечания

- Не хранить условия только в тексте `description` — они должны быть в полях/таблицах и доступны программно.
- Сохранить работу существующих списков и фильтров по вкладам (плоские поля или их авто-заполнение из новой структуры).
- Единицы измерения в БД зафиксировать: срок — в днях в `deposit_conditions`; суммы — в единицах выбранной валюты; ставка — годовой процент.
- При необходимости добавить справочник валют (например, отдельная таблица `currencies`) и ссылаться на неё из `deposit_currencies`; на первом этапе достаточно кода валюты строкой (RUB, USD, EUR).
